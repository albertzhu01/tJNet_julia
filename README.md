# tJNet_julia

Julia implementation of exact-diagonalization and neural-network variational tools
for finite t-J lattice models, done as part of MIT 18.337/6.7320 final project.

The repository is organized around two source modules:

- `tJ.jl`: lattice, basis, operator, and Hamiltonian construction.
- `tJNetRC.jl`: neural-network variational training, Monte Carlo sampling, and
  CPU/GPU execution helpers.

The notebooks in the repository provide tests, training sweeps, and profiling
workflows built on those two modules.

## Source Files

### `tJ.jl`

Defines the `TJModels` module. This file contains the lattice and many-body
basis machinery used by the rest of the project.

Main responsibilities:

- Construct finite periodic lattices through `Lattice`.
- Build t-J model sectors through `TJ` / `tJ`, with fixed spin-up, spin-down,
  and hole counts.
- Generate and index the Fock basis.
- Compute nearest and longer-range neighbors.
- Build spin, hole, hopping, and exchange operators.
- Assemble sparse Hamiltonian pieces such as hopping `Ht` and exchange `HJ`.
- Provide helpers for basis reordering, truncation, translations, and rotations.

This file uses Julia standard libraries such as `LinearAlgebra` and
`SparseArrays`.

### `tJNetRC.jl`

Defines the `TJNetRC` module. This file contains the neural-network training and
Monte Carlo code.

Main responsibilities:

- Build neural-network inputs from t-J basis states with `NN_inputs`.
- Define the Flux neural-network ansatz `NeuralNetwork`.
- Compute variational energies with `variational`.
- Convert Hamiltonians to CPU or GPU-compatible array formats with `processH`.
- Detect and initialize CUDA support with `gpu_available` and
  `cuda_device_count`.
- Train full-basis neural-network wavefunctions with `trainNN`.
- Train Monte Carlo sampled wavefunctions with `trainNNMC`.
- Generate Metropolis samples, estimate local energies, and evaluate MC
  observables.
- Save and load trained neural-network weights.

Required external packages include `Flux`, `Optim`, `Zygote`, and
`LineSearches`. GPU use is optional and requires `CUDA.jl`; training parameters
can request `"cpu"`, `"gpu"`, or `"auto"` devices.

## Notebooks

### `tJ_tests.ipynb`

Sanity-test notebook for the Julia implementation. It exercises lattice
construction, Hamiltonian assembly, exact variational calculations, Monte Carlo
sampling, and neural-network training on small systems. The cells are intended
to be run selectively while developing or validating changes.

### `tJ_training_sweep.ipynb`

Defines a parameter sweep over:

- site counts,
- hole counts,
- spin sectors,
- `J/t` values,
- CPU versus GPU training,
- full-basis versus Monte Carlo training.

The notebook is structured so expensive training calls can be launched from the
final cells after reviewing the configuration. It writes sweep outputs to
`training_sweep_results/`.

### `tJ_gpu_one_epoch_profile.ipynb`

Profiles Julia GPU full-basis neural-network training without Monte Carlo. It
breaks a training run into setup and per-epoch components, including:

- model and basis construction,
- Hamiltonian assembly,
- Hamiltonian transfer to GPU,
- input construction and transfer,
- neural-network forward pass,
- variational sparse-matrix energy calculation,
- Zygote backward pass,
- Flux optimizer update.

The notebook is useful for comparing isolated single-epoch timings with
sustained multi-epoch training behavior. It writes profiling rows to
`profile_results/gpu_one_epoch_profile.tsv`.

## Result Folders

### `profile_results/`

Contains tab-separated profiling outputs. Current files include:

- `gpu_one_epoch_profile.tsv`: Julia GPU profiling results.
- `gpu_one_epoch_profile_python.tsv`: corresponding Python/PyTorch profiling
  results used for comparison.

These files are generated artifacts and can be regenerated by rerunning the
profiling notebooks.

### `training_sweep_results/`

Contains tab-separated outputs from training sweeps and exact diagonalization.
Current files include:

- `exact_ground_energies.tsv`: exact ground-state energies from Hamiltonian
  diagonalization.
- `training_results_cpu_only_no_mc.tsv`: CPU full-basis training results.
- `training_results_gpu_only_no_mc.tsv`: GPU full-basis training results.
- `training_results_gpu_only_mc.tsv`: GPU Monte Carlo training results.
- `training_results_gpu_only_no_mc_python.tsv`: Python/PyTorch GPU full-basis
  training results used for comparison.

Large trained model files are not stored in the top-level repository layout
shown above, but sweep notebooks may create model subdirectories under
`training_sweep_results/` when model saving is enabled.
